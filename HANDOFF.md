# HANDOFF.md

## Короткая передача состояния

Проект сейчас находится в фазе точечной сборки публичной оболочки `Сарма Экспресс` вокруг уже существующего `SUPERBOX`.

Главное, что нужно помнить:

- `Сарма Экспресс` — это уже главная сайта
- `SUPERBOX` — это отдельное рабочее приложение на `/superbox`
- новая логика навигации строится через deep-link в `SUPERBOX flow`, а не через создание лишних страниц без необходимости

## На чем остановились

К моменту фиксации этого handoff уже сделано:

1. Главная `Сарма Экспресс` стоит на `/`
2. Есть отдельная страница калькулятора `/calculator`
3. Есть отдельная страница пунктов выдачи `/pickup-points`
4. Пункты выдачи уже отображаются на Яндекс Картах
5. Верхняя плашка `Сарма Экспресс` уже вынесена в общий sticky-компонент
6. Основные пункты меню уже связаны с рабочими сценариями `SUPERBOX`

## Самые важные файлы прямо сейчас

- `frontend/components/sarma-express-header.tsx`
- `frontend/components/sarma-express-page.tsx`
- `frontend/components/delivery-calculator-page.tsx`
- `frontend/components/pickup-points-page.tsx`
- `frontend/components/superbox-app.tsx`

## Что, скорее всего, будут просить дальше

По характеру последних задач от пользователя, наиболее вероятны следующие типы продолжения:

- еще одна привязка пунктов меню или кнопок к нужным `SUPERBOX flow`
- визуальная подгонка верхней плашки/отступов/размеров под новый скрин
- доработка калькулятора после того, как пользователь даст формулы
- доработка страницы пунктов выдачи или карты
- создание еще одной отдельной страницы в стиле `Сарма Экспресс`

## Что делать первым в новом диалоге

1. Прочитать `AGENT.md`, `CURRENT_STATUS.md`, `PROJECT_CONTEXT.md`
2. Проверить `git status --short`
3. Если задача про меню/шапку:
   - открыть `frontend/components/sarma-express-header.tsx`
4. Если задача про кнопку в `SUPERBOX`:
   - проверить существующие `flow` в `frontend/components/superbox-app.tsx`

## Важный нюанс по карте

Карта пунктов выдачи уже видна локально, но у Яндекса есть предупреждение `Invalid API key`. Это не нужно забывать при следующих задачах по карте.

Если пользователь скажет, что карта снова не работает, сначала проверять:

- `frontend/.env.local`
- `NEXT_PUBLIC_YANDEX_MAPS_API_KEY`
- ограничения ключа по доменам

## Чего не делать без явной просьбы

- не объединять полностью `Сарма Экспресс` и `SUPERBOX`
- не переписывать `SUPERBOX` под стиль `Сарма Экспресс`
- не трогать backend и Bitrix, если задача про UI и навигацию
- не откатывать существующие незакоммиченные изменения
